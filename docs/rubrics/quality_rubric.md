# Quality Rubric

最终结果需要经过 ReviewerAgent 和最终质量 Hook 检查。

## 必要检查项

| 检查项 | 通过条件 |
|---|---|
| 原始数据存在 | 至少保存 10 条 raw items |
| 结构化记录存在 | 至少保存 10 条 structured records |
| Schema 完整性 | 必填字段齐全 |
| 来源可追踪 | 每条记录包含来源 URL 或 raw reference |
| 报告完整性 | 必要章节都存在 |
| 可视化存在 | 至少有一个图表产物 |
| 分析有支撑 | 热点和趋势能引用结构化记录 |
| 不是一次性生成 | 中间产物能证明分阶段处理 |

## ReviewerAgent 关注点

ReviewerAgent 检查语义质量：

- Top 事件是否合理？
- 影响分析是否具体？
- 趋势判断是否由多条记录支撑？
- 风险/机会提示是否有依据？
- 报告是否避免了空洞套话？

## 最终质量 Hook 关注点

最终质量 Hook 检查机械完整性：

- 必要文件是否存在
- 必填字段是否存在
- 数据条数是否达标
- JSON 是否合法
- Markdown 标题是否齐全
- 图表产物是否存在
